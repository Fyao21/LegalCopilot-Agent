# ADR-0001：先使用 SQLite

- 状态：已采纳
- 日期：2026-07-23

## 背景

求职演示项目需要在新电脑上快速启动。如果第一版强制依赖 MySQL 或 PostgreSQL，评审者必须先安装服务、创建账号和导入数据，失败点会显著增加。

## 决策

默认使用项目内 SQLite 文件，启动时自动建表并导入教学法规。SQLAlchemy 隔离数据库访问，使业务代码不绑定 SQLite 方言。

## 后果

优点是零外部依赖、数据库文件可直接检查、测试容易使用内存数据库。代价是并发写入能力有限、缺少原生向量索引。后台任务对 SQLite `database is locked` 增加有限重试，但它不是高并发方案。

## 迁移条件

出现多实例部署、多用户并发写、法规达到十万级、需要权限审计或向量索引时迁移 PostgreSQL。
