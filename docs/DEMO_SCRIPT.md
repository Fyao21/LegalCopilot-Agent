# 2–3 分钟演示脚本

## 录制前准备

1. 后端和前端均已启动，浏览器缩放为 100%。
2. 使用脱敏样例：“公司拖欠三个月工资，并且一直没有签订书面劳动合同。”
3. 准备一个离线运行结果和一个成功的 Agent 运行结果。
4. 关闭包含 API Key、PyCharm 环境变量和私人通知的窗口。

## 分镜与讲解

### 0:00–0:20 项目问题

展示首页并说明：直接让模型回答法律问题可能虚构法条。本项目将案件抽取、法规检索、引用审核和报告生成拆成可追踪节点，所有引用都能回查数据库 ID。

### 0:20–0:50 提交案件

粘贴脱敏问题，切换“智能 Agent”，点击分析。指出接口立即返回任务 ID，长任务在后台执行，页面轮询状态。

### 0:50–1:20 查看轨迹

展示四个节点、耗时和执行引擎标签。说明 `智能 Agent · 模型名` 表示真实调用；`Agent 已降级` 表示模型不可用后转入规则方案。

### 1:20–1:50 报告与引用

打开报告和引用抽屉，展示 article ID、数据库原文、关键词/语义分数及审核理由。强调模型不能把自创法条写入报告。

### 1:50–2:15 评测

展示 `eval/results/latest.md`：24 条脱敏数据、案件类型准确率、Recall@5、MRR、工作流成功率。主动说明当前小数据集上关键词优于哈希向量，真实 Embedding 组尚需独立服务。

### 2:15–2:40 工程质量

展示 GitHub Actions：Ruff、mypy、32 项测试、离线评测和前端构建。说明 CI 不读取本机 `.env`，所有模型调用均被禁用或 Mock。

### 2:40–3:00 收尾

展示架构图和限制：教学法规不等于权威法规、后台任务尚未迁移持久队列、生产环境需要 PostgreSQL/pgvector 和鉴权。

## 验收清单

- [ ] 视频没有出现 API Key、个人姓名或私人通知。
- [ ] 能看清模式选择、运行轨迹、引用详情和评测指标。
- [ ] 时长不超过 3 分钟。
- [ ] 上传到仓库 README 可访问的位置并替换演示占位说明。
